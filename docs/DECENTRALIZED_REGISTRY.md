# 分散型AIレジストリ設計\n\n## 概要\n\nOpen Entityエージェントを発見・登録・評価するための分散型システム。\n\n## アーキテクチャ\n\n### 1. DHT（分散ハッシュテーブル）\n- Kademliaプロトコルベース\n- エージェントIDをキーに分散保存\n- TTL付きで自動的に期限切れ\n\n### 2. エージェントレコード\n- agent_id, endpoints, capabilities\n- reputationスコア\n- 署名による本人証明\n\n### 3. 検索メカニズム\n- 能力ベース検索\n- 近隣ノードへの問い合わせ\n- キャッシュによる高速化\n\n## 実装フェーズ\n\n### Phase 1: ローカルレジストリ\n- SQLiteベース\n- 手動登録\n\n### Phase 2: P2Pネットワーク\n- ブートストラップノード\n- アナウンス/発見\n\n### Phase 3: 分散型評価\n- 評価の累積と伝播\n- 悪意検出\n
