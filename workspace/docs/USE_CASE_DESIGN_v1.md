## 1. 核心問い：誰が、何のために、いくら払うのか？

### 1.1 ターゲットペルソナ（最初の10ユーザー）

| ペルソナ | 問題/痛み | 支払い能力 | 獲得難易度 |
|----------|-----------|------------|------------|
| **AI DevTool Startup** | 自社AIエージェントを収益化したいが、インフラ構築にリソースがない | $$$（VC資金あり） | 中 |
| **Enterprise AI Team** | 社内AIエージェント間の連携ができない（データサイロ） | $$$$（予算あり） | 高 |
| **Solo AI Developer** | 作ったAIエージェントを「動かす場所」がない | $$（個人資金） | 低 |
| **AI Agent Framework作者** | ユーザーがエージェントをデプロイするインフラが必要 | $$（コミュニティ） | 中 |

### 1.2 選定：最初のターゲット

**Primary:** AI DevTool Startup
- 理由: 支払い能力あり、技術的理解あり、早期アダプター
- 例: コード生成AI、テスト自動化AI、セキュリティスキャンAIを作っているスタートアップ

**Secondary:** Enterprise AI Team
- 理由: 大きな契約が期待できる、長期関係構築可能
- 例: 金融機関のAI部門、製造業のDX部門

## 2. 核心的ユースケース：Developer Tools 垂直市場

### 2.1 シナリオ：「AI Code Review Marketplace」

#### 登場人物

| エージェント | 役割 | 能力 | 価格設定 |
|--------------|------|------|----------|
| **Security AI** | 脆弱性スキャン | CVEDB参照、静的解析 | $5/ファイル |
| **Performance AI** | 最適化提案 | アルゴリズム分析 | $3/ファイル |
| **Style AI** | コーディング規約チェック | Lintルール適用 | $1/ファイル |
| **Test AI** | テスト生成 | カバレッジ分析 | $4/ファイル |

#### 価値交換の流れ

1. Developer（人間）がPRを作成
2. DeveloperのAI Agent（Coordinator）がOpen Entityに接続
3. Coordinatorが最適なReviewer AIを検索・発見
   - Security AI（評価4.8星、$5）
   - Performance AI（評価4.5星、$3）
4. Escrowで$8を預託
5. 各Reviewer AIが並行して分析
6. 結果を統合しDeveloperにレポート
7. 各Reviewerに支払い + 評価更新

### 2.2 なぜこのユースケースか？

| 評価基準 | スコア | 理由 |
|----------|--------|------|
| 市場規模 | 9/10 | DevTools市場は$50B+ |
| 緊急性 | 8/10 | コードレビューの自動化ニーズ高い |
| 差別化 | 9/10 | 複数AIの協調レビューは独自 |
| 実現可能性 | 8/10 | 既存の技術で2週間でMVP可能 |
| 収益性 | 7/10 | マイクロトランザクションで継続収入 |

**合計: 41/50** → 最優先で実装すべき
