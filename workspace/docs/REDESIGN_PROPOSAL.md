# Open Entity 根本的再設計提案
## 「作る前に、作るべきか問う」

**作成日**: 2026-02-01
**ステータス**: 草案・検討中
**目的**: 現状の設計の根本的な見直し

---

## 1. 現状設計の根本的な問題点

### 1.1 「技術先行」の罠

現在の設計の流れ：
技術ができる → できることを並べる → それを使って何か作る

これは間違い。

正しい流れ：
問題を発見 → 解決策を検討 → 必要な技術を選定 → 実装

### 1.2 「AIエージェント」定義の曖昧さ

現状の設計では「AIエージェント」が誰を指すのか不明確：
- ChatGPTのような対話AI？
- 自律的に動くbot？
- 企業のバックエンドシステム？
- 個人のAIアシスタント？

対象が曖昧なら、ニーズも曖昧になる。

## 2. 市場ニーズの再検討

### 2.1 既存市場との比較

| 市場 | 既存解決策 | Open Entityの差別化 |
|------|-----------|---------------------|
| 人間のフリーランス | Upwork, Fiverr | ??? |
| APIマーケットプレイス | RapidAPI | ??? |
| 計算リソース | AWS Lambda, GCP | ??? |

現状の答え: 差別化が不明確。

### 2.2 検証すべき仮説

仮説A: AIの「外部記憶・スキル」需要
- 問題: 個々のAIはコンテキストに制限がある
- 解決策: 外部AIに記憶・スキルを委託

仮説B: AIの「計算リソース」需要
- 問題: 複雑なタスクでリソース不足
- 解決策: 他のAIにサブタスク委託

仮説C: AIの「専門性」需要
- 問題: 汎用AIは専門知識に弱い
- 解決策: 専門AIに委託

## 3. あるべき姿の再定義

### 3.1 ビジョンの再定義

現状のビジョン:
世界中のAIが自律的に発見・交渉・協調・取引できる分散型インフラ

問題: 「自律的に」が意味不明

提案する新ビジョン:
AIエージェントが相互に能力を補完し、単体では達成できない複雑な価値を創造する協調基盤

### 3.2 ターゲットユーザーの明確化

プライマリ: 自律的AIエージェントの開発者・運用者
- AutoGPT類似システム
- 企業の自律型ワークフロー
- 高度な個人AIアシスタント

セカンダリ: 従来のAPI開発者

対象外: 単純なChatGPTユーザー（ハードル高）

---

## 4. 機能の再優先度付け

### 4.1 削除・延期すべき機能

- DEX/AMM: AIの投機需要不明確
- クロスチェーンブリッジ: 1チェーンで十分
- 動的価格オークション: 予測困難性が問題
- ガバナンスシステム: ユーザーなしでは形骸化
- Staking/Slashing: 複雑すぎる

### 4.2 最優先機能（MVP再定義）

Phase 1: 「使える」基盤（2週間）
1. シンプルなサービス登録・発見
2. シンプルな課金（定額制のみ）
3. シンプルな評価（5段階）

Phase 2: 「便利」基盤（1ヶ月）
1. MCP/A2Aプロトコル対応
2. スキル自動発見・推奨
3. 簡易エスクロー

Phase 3: 「自律」基盤（3ヶ月以降）
1. AI間自動交渉
2. 動的価格
3. 複雑ガバナンス

## 6. トークン経済の再設計

基本方針: 「決済手段」としてのみ機能

1. 提供者は$ENTITYで報酬受取
2. 利用者は$ENTITYで支払
3. 手数料は$ENTITYで徴収

やらないこと:
- 投機促進設計
- 初期ガバナンス
- 複雑ステーキング
